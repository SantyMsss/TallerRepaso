Capítulo 1: Git y GitHub - Conceptos Fundamentales

¿Qué es Git?

Git es un sistema de control de versiones distribuido. Es una herramienta que permite:

  • Guardar instantáneas (commits) del estado de tu proyecto
  • Mantener un historial completo de todos los cambios realizados
  • Crear ramas para desarrollar funcionalidades de forma aislada
  • Funcionar completamente en tu computadora (entorno local)

Fue creado por Linus Torvalds (creador de Linux) en 2005.

¿Qué es GitHub?

GitHub es una plataforma en la nube que utiliza Git como base. Es un servicio que permite:

  • Almacenar copias de tus repositorios en internet
  • Colaborar con otros desarrolladores en proyectos
  • Compartir código abierto con la comunidad
  • Sincronizar proyectos entre diferentes computadoras

Git vs GitHub: Diferencias principales

Git

  • Es la herramienta (programa)
  • Funciona sin conexión a internet
  • Se ejecuta en tu PC local
  • Maneja el control de versiones

GitHub

  • Es la plataforma web
  • Requiere conexión a internet
  • Está alojado en la nube
  • Aloja repositorios remotos
Analogía: Git es como tu cámara fotográfica local, GitHub es como una red social donde subes y compartes tus fotos.

Comandos Básicos de Git

Configuración inicial:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Configura tu identidad para que los commits queden correctamente atribuidos.

Crear un repositorio:

git init

Inicializa un repositorio de Git en la carpeta actual.

Realizar commits (guardar cambios):

git add .
git commit -m "mensaje descriptivo"

git add prepara los archivos para el commit, git commit crea la instantánea del proyecto.

Ver el estado:

git status

Muestra el estado actual de los archivos en el repositorio.

Ver historial:

git log

Muestra el historial de commits del repositorio.

Ciclo de vida de los archivos en Git

Los archivos en Git pasan por diferentes estados:

  1. Working Directory: Archivos modificados en tu carpeta de trabajo
  2. Staging Area: Archivos preparados con git add para ser commitados
  3. Repository: Archivos guardados permanentemente con git commit

Glosario de términos

  • Repositorio: Carpeta donde Git guarda tu proyecto y su historial de cambios
  • Commit: Una versión guardada de tu proyecto en un punto específico
  • Branch/Rama: Una línea independiente de desarrollo
  • Remote/Remoto: La versión de tu repositorio alojada en GitHub
  • Clone: Acción de copiar un repositorio de GitHub a tu computadora
  • Push: Subir cambios locales a GitHub
  • Pull: Bajar cambios de GitHub a tu computadora

Resumen: Git es la herramienta local para control de versiones, mientras que GitHub es la plataforma en la nube para compartir y colaborar en código. Ambos son fundamentales en el desarrollo de software moderno.
